# Quiz: Królowie Polski — koronacje, dynastia, kontekst

## Cel quizu

### Cel główny (primary outcome)

> Uczeń potrafi przypisać większości koronowanych władców Polski poprawny **rok koronacji**.

Głównym mierzalnym efektem jest zapamiętanie roku. Pełna data (dzień, miesiąc) traktowana jest jako ciekawostka / poziom rozszerzony — nie jest wymagana w żadnym z trybów podstawowych.

### Cele wspierające

Poniższe pomagają budować skojarzenia i osadzać daty w kontekście, ale nie są samodzielnym celem quizu:

- przyporządkowanie władcy do dynastii
- znajomość kolejności chronologicznej
- powiązanie władcy z kluczowym wydarzeniem panowania

**Grupa docelowa:** klasa 6 SP, uczeń zainteresowany historią.

## Kontekst dydaktyczny — co wyświetlać przy każdym królu

Aby budować skojarzenia (elaborative encoding), każdy władca ma przypisane pola kontekstowe. Wyświetlamy je **po odpowiedzi** jako „kartę wiedzy" — w dwóch warstwach.

### Pola danych na władcę

```
{
  name:          "Władysław II Jagiełło",
  coronationYear: 1386,
  coronationFull: "4 marca 1386",
  place:         "Kraków",
  dynasty:       "Jagiellonowie",
  predecessor:   "Jadwiga Andegaweńska",
  successor:     "Władysław III Warneńczyk",
  keyAssociation: "Bitwa pod Grunwaldem (1410) — klęska Krzyżaków",
  keyEvents: [
    "Unia w Krewie (1385) — połączenie Polski i Litwy",
    "Bitwa pod Grunwaldem (1410) — klęska Krzyżaków",
    "Odnowienie Akademii Krakowskiej (1400)"
  ],
  funFact:       "Przed koronacją był wielkim księciem litewskim i nosił imię Jogaila. Chrzest przyjął zaledwie 17 dni przed koronacją.",
  mnemonic:      "Jagiełło ↔ Grunwald 1410 — najsłynniejsza polska bitwa"
}
```

### Karta wiedzy — dwie warstwy

**Warstwa podstawowa** (wyświetlana zawsze po odpowiedzi):

```
┌───────────────────────────────────────────────┐
│  ✓ Dobrze!  Władysław II Jagiełło — 1386     │
│                                               │
│  Dynastia:    Jagiellonowie                   │
│  Skojarzenie: Bitwa pod Grunwaldem (1410)     │
│                                               │
│             [Dalej]    [Więcej ▼]             │
└───────────────────────────────────────────────┘
```

Zawartość minimalna:
- poprawna odpowiedź (rok)
- dynastia
- **jedno** najmocniejsze skojarzenie (`keyAssociation`)

**Warstwa rozszerzona** (po kliknięciu „Więcej"):

```
┌───────────────────────────────────────────────────────┐
│  ✓ Dobrze!  Władysław II Jagiełło — 4 marca 1386     │
│                                                       │
│  Dynastia:     Jagiellonowie                          │
│  Poprzednik:   Jadwiga Andegaweńska                   │
│  Następca:     Władysław III Warneńczyk               │
│                                                       │
│  ► Unia w Krewie (1385) — połączenie Polski i Litwy   │
│  ► Bitwa pod Grunwaldem (1410)                        │
│  ► Odnowienie Akademii Krakowskiej (1400)             │
│                                                       │
│  💡 Przed koronacją był wielkim księciem litewskim     │
│     — nosił imię Jogaila.                             │
│                                                       │
│                    [Dalej]                             │
└───────────────────────────────────────────────────────┘
```

W warstwie rozszerzonej: pełna data, poprzednik/następca, wszystkie wydarzenia, ciekawostka. Uczeń sam decyduje, kiedy chce więcej.

> **Nawigacja:** przejście do następnego pytania sterowane przyciskiem „Dalej" (nie stałym timerem). Uczeń kontroluje tempo.

### Skrócony opis kontekstowy — wszystkie dynastie

**Piastowie (1025–1370)** — pierwsza polska dynastia; od Chrobrego przez rozbicie dzielnicowe do zjednoczenia przez Łokietka i złotego wieku Kazimierza Wielkiego.

**Andegawenowie (1370–1399)** — dwaj władcy z Węgier; Jadwiga — jedyna kobieta koronowana jako „król" (rex) Polski.

**Jagiellonowie (1386–1572)** — najdłuższa dynastia; unia z Litwą, złoty wiek kultury, ostatni Jagiellon (Zygmunt August) umarł bezpotomnie → wolna elekcja.

**Władcy elekcyjni (1573–1795)** — od Walezego do Poniatowskiego; szlachta wybiera króla; Sobieski pod Wiedniem; rozbiory kończą Rzeczpospolitą.

## Tryby quizu

Quiz oferuje **4 tryby gry**. Wspólna mechanika: streak 3× z rzędu do zaliczenia. **Tryby nie są dostępne od razu** — odblokowują się w miarę postępów (patrz: Progresja uczenia).

### Tryb 1: Król → Rok koronacji (wpisywanie)

Wyświetlamy **tylko imię króla**. Uczeń wpisuje rok koronacji. Dynastia i panowanie **nie są** pokazywane w prompt — żeby nie zdradzać odpowiedzi (rok koronacji często pokrywa się z początkiem panowania).

```
┌─────────────────────────────────────┐
│                                     │
│    Władysław II Jagiełło            │
│                                     │
│    Rok koronacji: [____]  [Sprawdź] │
│                                     │
│    Seria: ●●○                       │
└─────────────────────────────────────┘
```

Opcjonalny przełącznik „Pokaż wskazówki" (scaffolding) — dodaje dynastię. Domyślnie wyłączony.

### Tryb 2: Rok → Król (wybór z 4 opcji)

Wyświetlamy rok koronacji. Uczeń wybiera właściwego króla spośród 4 propozycji.

```
┌─────────────────────────────────────┐
│    Koronacja: 1025 rok              │
│                                     │
│    ○ Władysław Łokietek             │
│    ○ Bolesław I Chrobry             │
│    ○ Mieszko II Lambert             │
│    ○ Kazimierz III Wielki           │
└─────────────────────────────────────┘
```

Dystraktory dobierane inteligentnie — z tej samej dynastii lub bliskich dat.

### Tryb 3: Oś czasu (przeciągnij i upuść)

Losowanych **4–6 królów** — uczeń ustawia ich w kolejności chronologicznej. Rola tego trybu: **budowanie relacji i porządku**, nie trening precyzyjnych dat.

```
┌────────────────────────────────────────────────────┐
│  Ułóż królów w kolejności koronacji:               │
│                                                    │
│  ╔════════════╗  ╔═══════════╗  ╔════════════════╗ │
│  ║ Jagiełło   ║  ║ Łokietek  ║  ║ Kaz. Wielki    ║ │
│  ╚════════════╝  ╚═══════════╝  ╚════════════════╝ │
│  ╔════════════╗                                    │
│  ║ Chrobry    ║                                    │
│  ╚════════════╝                                    │
│                                                    │
│  [___] → [___] → [___] → [___]                    │
│  wcześniej                później                  │
└────────────────────────────────────────────────────┘
```

### Tryb 4: Kontekst → Król (kto to?)

Wyświetlamy opis/wydarzenie. Uczeń rozpoznaje króla. **Ważne:** wskazówki powinny być zróżnicowane — nie tylko najbardziej ikoniczne skojarzenia, ale też relacje dynastyczne, miejsce w sekwencji, charakterystyka epoki.

Typy wskazówek (mieszane losowo):

| Typ wskazówki | Przykład |
|---------------|----------|
| Wydarzenie | „Za jego panowania doszło do bitwy pod Grunwaldem (1410)." |
| Relacja dynastyczna | „Następca Jadwigi Andegaweńskiej. Pierwszy Jagiellon na tronie." |
| Miejsce w sekwencji | „Koronowany po Łokietku. Ostatni Piast na polskim tronie." |
| Opis panowania | „Zastał Polskę drewnianą, a zostawił murowaną. Panował 37 lat." |
| 2 umiarkowane wskazówki | „Władca elekcyjny. Poprowadził odsiecz wiedeńską." |

```
┌──────────────────────────────────────────────────────┐
│  Następca Jadwigi Andegaweńskiej.                    │
│  Pierwszy Jagiellon na polskim tronie.               │
│                                                      │
│  Który to król? [________________________] [Sprawdź] │
└──────────────────────────────────────────────────────┘
```

## Elementy budujące skojarzenia (metodyka)

| Technika | Realizacja w quizie |
|----------|---------------------|
| **Elaborative encoding** | Karta wiedzy z kontekstem po każdej odpowiedzi — uczeń powiązuje rok z wydarzeniem i dynastią |
| **Mastery loop** (utrwalanie w sesji) | Streak 3× z rzędu; błędna odpowiedź resetuje serię |
| **Intra-session spacing** | Pytania błędne wracają po 3–5 innych pytaniach; trudne (błędne ≥2×) po 8–12; opanowane rzadziej |
| **Interleaving** | Miks trybów — ale dopiero po opanowaniu podstaw (patrz: Progresja) |
| **Retrieval practice** | Uczeń aktywnie przypomina sobie odpowiedź; prompt nie zawiera podpowiedzi zdradzającej rok |
| **Dual coding** | Oś czasu (wizualna) + tekst; karta wiedzy łączy fakty z narracją |
| **Chunking** | Podział na dynastie — domyślnie mała paczka na start |
| **Mnemoniki** | Pole `mnemonic` — krótkie zdanie-skojarzenie (warstwa rozszerzona karty wiedzy) |

> **Uwaga dot. nazewnictwa:** mechanizm streaku i powrotów trudnych pytań to **utrwalanie w obrębie sesji** (mastery loop + intra-session spacing), nie pełny system spaced repetition między sesjami. Ewentualne międzysesyjne powtórki to potencjalne rozszerzenie.

## Progresja uczenia

Kluczowa zmiana vs. pierwotny plan: **quiz nie zaczyna od najtrudniejszego trybu** (free recall = wpisywanie roku). Zamiast tego prowadzi ucznia przez progresję trudności.

### Ścieżka dla nowego ucznia

| Etap | Tryb | Zakres | Cel |
|------|------|--------|-----|
| 1 | **Rozpoznawanie** (Tryb 2: rok → wybierz króla z 4 opcji) | 1 paczka dynastyczna (5–7 postaci) | Pierwszy kontakt z materiałem; niski próg wejścia |
| 2 | **Porządkowanie** (Tryb 3: oś czasu) | Ta sama paczka | Budowanie porządku chronologicznego |
| 3 | **Przypominanie** (Tryb 1: król → wpisz rok) | Ta sama paczka; bez panowania w prompt | Aktywny recall — najtrudniejszy tryb |
| 4 | **Kontekst** (Tryb 4: opis → rozpoznaj króla) | Ta sama paczka | Utrwalenie przez bogate skojarzenia |
| 5 | **Miks trybów** | Ta sama paczka, losowany tryb | Interleaving — wzmocnienie retencji |
| 6 | **Kolejna paczka** | Następna dynastia | Powrót do etapu 1 z nowym materiałem |
| 7 | **Powtórka całości** | Wszyscy (28) | Konsolidacja; tryb zaawansowany |

> Uczeń może pominąć etapy i od razu wybrać dowolny tryb / zakres — progresja to **sugerowana ścieżka**, nie blokada.

### Intra-session spacing (powroty trudnych pytań)

Zamiast czysto losowego doboru pytań, pytania wracają z częstotliwością zależną od wyniku:

| Status pytania | Powraca po… |
|----------------|-------------|
| Błędne (1. raz) | 3–5 innych pytaniach |
| Błędne (≥2 razy) | 8–12 innych pytaniach |
| Poprawne (seria < 3) | losowo, w normalnej rotacji |
| Opanowane (seria = 3) | nie wraca w tej sesji |

### Pipeline ekstrakcji danych

Skrypt `scripts/extract_borders.py` (Python, venv via `uv`):

1. Pobranie GeoJSON z aourednik dla 11 lat źródłowych
2. Wyodrębnienie wielokąta Polski z każdego pliku
3. Dla lat 1400+: rozdzielenie Korona/Litwa przybliżoną linią graniczną (shapely)
4. Zapis jako `borders_data.json` — lekki plik z koordynatami per era

```bash
uv venv
uv pip install shapely requests
python scripts/extract_borders.py
```

Wynikowy `borders_data.json` zostaje wcommitowany do repo — quiz ładuje go bez potrzeby uruchamiania Pythona.

### Wytyczne graficzne

- **Bounding box:** stały dla wszystkich er (~14°E–34°E, ~49°N–57°N) — Europa Środkowo-Wschodnia

- **Etykieta:** pod mapą, `font-size: 0.75rem`, kolor `--text-muted`
- **Rendering:** `<svg>` z `<path>` — lekkie, skalowalne

## Technologia

- Samodzielny `index.html` (jak wszystkie quizy w projekcie)
- Zero zewnętrznych zależności w runtimie (czysty HTML/CSS/JS)
- Dane wszystkich 28 królów jako tablica JS w pliku
- Dane granic jako osobny `borders_data.json` (ładowany fetch) lub inline
- Drag & drop (tryb osi czasu) — natywne HTML5 Drag and Drop API
- Responsywny — działa na tablecie (typowe urządzenie w klasie)
- Pipeline ekstrakcji: Python + uv (shapely, requests) — jednorazowy, wynik commitowany

---

## Kolejność implementacji

1. **Faza 1 — MVP dydaktyczne**
   - Tryb 2 (rok → wybierz króla z 4 opcji) — prostszy tryb na start
   - Karta wiedzy (warstwa podstawowa: rok + dynastia + 1 skojarzenie)
   - Filtrowanie po dynastii (domyślnie: Piastowie)
   - Intra-session spacing (trudne pytania wracają częściej)

2. **Faza 2 — Aktywne przypominanie**
   - Tryb 1 (król → wpisz rok) — bez panowania w prompt
   - Przełącznik „Pokaż wskazówki" (scaffolding: dynastia)
   - Warstwa rozszerzona karty wiedzy („Więcej")

3. **Faza 3 — Mapy historycznych granic**
   - Pipeline ekstrakcji (`scripts/extract_borders.py`, uv + shapely)
   - `borders_data.json` — 11 er z koordynatami Korona/Litwa
   - Rendering SVG na karcie wiedzy
   - Pole `borderEra` w danych każdego króla

4. **Faza 4 — Budowanie relacji**
   - Tryb 3 (oś czasu / drag & drop, 4–6 postaci)
   - Presetowe paczki po 5–7 postaci

5. **Faza 5 — Utrwalanie przez kontekst**
   - Tryb 4 (kontekst → król) z różnorodnymi typami wskazówek
   - Miks trybów (interleaving po opanowaniu paczki)

6. **Faza 6 — Podsumowanie i integracja**
   - Ekran końcowy z listą najtrudniejszych
   - Sugerowana ścieżka progresji na ekranie startowym
   - Kafelek na landing page
